## หลักการออกแบบหน้าเลือกหมวด

- `index.html` เป็นหน้าศูนย์รวม ไม่ควรเอาเกมใหม่ไปปนในหน้าอังกฤษโดยตรง
- ถ้ามีวิชาใหม่ เช่น ภาษาไทย ให้สร้างโฟลเดอร์ใหม่ เช่น `thai-quiz/index.html`
- เพิ่มการ์ดลิงก์ใน `index.html` ให้แยกหมวดชัดเจน
- ลิงก์ GitHub Pages จะเป็นรูปแบบ `https://ditthakornm-star.github.io/eng-word-game/ชื่อโฟลเดอร์/`

## หลักการออกแบบเกมคณิต

ไฟล์หลักคือ `math-quiz/index.html`

แนวทางที่ตกลงไว้:
- มีหน้าหลักให้เลือก เล่นรวม 60 ข้อ หรือเลือกหน่วย 1-4
- ทุกครั้งที่เริ่มเล่น ต้องสุ่มโจทย์และตัวเลือกใหม่
- หน่วยละ 15 ข้อ รวม 60 ข้อ
- โจทย์ต้องยากพอสำหรับซ้อมสอบ ไม่ใช้โจทย์ง่ายเกินไป เช่น `0+0`, `10+10`, `9-1`, `8-1`
- อย่าให้รูปหรือโจทย์มีเฉลยแอบอยู่ เช่น ไฮไลต์คำตอบในภาพ
- ใช้อีโมจิสิ่งของ/อาหาร/เครื่องเขียน แทนหน้าเด็กหรือสัตว์
- ไม่มีเสียงอ่านโจทย์ เพราะ browser อ่านคำไทยอย่าง “บวก/ลบ” เพี้ยน
- เหลือเฉพาะเสียงเอฟเฟกต์ตอนตอบถูกและตอบผิด

หน่วยที่ใช้ตอนนี้:
- หน่วย 1: จำนวนนับ 1-10 และ 0
- หน่วย 2: การบวกที่ผลบวกไม่เกิน 10
- หน่วย 3: การลบที่ตัวตั้งไม่เกิน 10
- หน่วย 4: จำนวนนับ 11-20

เวลาจะเพิ่มความยาก:
- เพิ่มรูปแบบคำถามใน function สร้างโจทย์ของแต่ละหน่วย
- ระวังตัวเลือกหลอก ต้องมีคำตอบถูกเพียงข้อเดียว
- สำหรับข้อเปรียบเทียบ ให้ถามแบบเจาะจง เช่น “จำนวนใดน้อยกว่า 20 อยู่ 1” แทน “จำนวนใดน้อยกว่า 20”

## หลักการออกแบบเกมคำศัพท์อังกฤษ

ไฟล์หลักของแต่ละชุดคือ:
- `week-july-6/index.html`
- `week-last/index.html`

แนวทางที่ตกลงไว้:
- มีหน้าเลือกชุดคำศัพท์/สัปดาห์
- มีปุ่มกลับหน้าเริ่มต้น
- โหมดง่ายมาก: ให้ดูคำเต็มใน popup ก่อน กด “พร้อม” แล้วค่อยพิมพ์
- โหมดง่าย: มีคำใบ้ กดได้ไม่เกิน 2 ครั้งต่อคำ
- โหมดยาก: ไม่ให้คำใบ้
- คำยาวควรแบ่งเป็นช่วงให้จำง่าย เช่น `fibrous root` แยกเป็น `fi-brous root` หรือแยกตามจังหวะอ่านที่เด็กจำได้
- ชุดเก่าควรยังอยู่ เพื่อให้เด็กกลับไปทวนได้

## วิธีอัปเดตขึ้น GitHub Pages

แก้ไฟล์ใน repo นี้ก่อน:


จากนั้นเช็กและอัปขึ้น GitHub:

```powershell
git status --short
git add .
git commit -m "คำอธิบายสั้นๆ ว่าแก้อะไร"
git push origin main
```

หลัง push แล้ว GitHub Pages มักใช้เวลาประมาณ 30-90 วินาทีจึงจะเห็นผล

ถ้ามือถือยังเห็นหน้าเก่า:
- ปิดแท็บแล้วเปิดใหม่
- หรือเติมท้ายลิงก์ เช่น `?v=ใหม่`
- หรือ clear cache ใน browser

## วิธีเพิ่มบทใหม่หรือวิชาใหม่

1. สร้างโฟลเดอร์ใหม่ เช่น `thai-quiz/` หรือ `math-unit-5/`
2. ใส่ไฟล์ `index.html` ในโฟลเดอร์นั้น
3. เพิ่มปุ่มลิงก์ในหน้า `index.html` หลัก
4. ทดสอบในเครื่องก่อน
5. commit และ push
6. เปิดลิงก์ GitHub Pages เพื่อตรวจจริง
