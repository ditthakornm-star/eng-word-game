## หลักการออกแบบเกมคณิต

ไฟล์หลักคือ `ป.1/คณิต/math-quiz-app/index.html` (บน GitHub Pages ใช้ชื่อโฟลเดอร์ `math-quiz/`)

แนวทางที่ตกลงไว้:
- มีหน้าหลักให้เลือก เล่นรวมทุกหน่วย หรือเลือกทีละหน่วย 1-7
- โหมดรวม: สุ่มหน่วยละ 6 ข้อ รวม 42 ข้อ / โหมดหน่วยเดียว: 12 ข้อ
- ทุกครั้งที่เริ่มเล่น ต้องสุ่มโจทย์และช่องตอบใหม่
- โจทย์ต้องยากพอสำหรับซ้อมสอบ เลี่ยงโจทย์ที่เดาได้ทันที เช่น `6+0`, `9-1`, “จำนวนถัดจาก 19”
- เรื่องศูนย์ให้ถามเชิงเหตุผลแทน เช่น `8 - ⬜ = 8` หรือ `⬜ - 6 = 0` ไม่ใช่ `6-0` ตรงๆ
- เน้นแนวข้อสอบ: เติมค่าแบบคิดย้อน (`⬜ - 5 = 7`), ลำดับเว้นช่อง/นับทีละ 2, เทียบผลบวกสองข้าง, บวกทดข้ามสิบ, โจทย์ปัญหาสองขั้นตอน
- อย่าให้รูปหรือโจทย์มีเฉลยแอบอยู่ เช่น ไฮไลต์คำตอบในภาพ
- ใช้อีโมจิสิ่งของ/อาหาร/เครื่องเขียน แทนหน้าเด็กหรือสัตว์
- ไม่มีเสียงอ่านโจทย์ เพราะ browser อ่านคำไทยอย่าง “บวก/ลบ” เพี้ยน
- เหลือเฉพาะเสียงเอฟเฟกต์ตอนแตะ ตอบถูก และตอบผิด

หน่วยครบตามหนังสือเรียน (7 หน่วย):
- หน่วย 1: จำนวนนับ 1-10 และ 0
- หน่วย 2: การบวกที่ผลบวกไม่เกิน 10
- หน่วย 3: การลบที่ตัวตั้งไม่เกิน 10
- หน่วย 4: จำนวนนับ 11-20
- หน่วย 5: การบวกที่ผลบวกไม่เกิน 20 (มีบวกสามจำนวน)
- หน่วย 6: การลบที่ตัวตั้งไม่เกิน 20
- หน่วย 7: รูปเรขาคณิต 2 มิติ และ 3 มิติ (วาดด้วย SVG ในโค้ด)

รูปแบบการเล่น (input.type) ที่เพิ่มมาให้เหมาะกับจอสัมผัส — ดัดแปลงจากแบบฝึกหัดจริง:
- `pad` แผงตัวเลข 0-10 หรือ 0-20 ให้แตะตอบ (แทนช่องเขียนในสมุด)
- `compare` ปุ่ม `>` `<` `=` ขนาดใหญ่
- `operator` ปุ่ม `+` `-` (เติมเครื่องหมายให้ได้คำตอบ)
- `tenframe` แตะช่องให้ครบตามจำนวน (ระบายให้ครบ)
- `order` แตะการ์ดเรียงลำดับน้อย→มาก หรือ มาก→น้อย
- `bondmake` แตะสองจำนวนที่รวมกันได้ตามส่วนรวม (ตอบได้หลายแบบ)
- `choice` เลือกตอบ ใช้กับชื่อรูปเรขาคณิต/เลือกประโยคสัญลักษณ์

เวลาจะเพิ่มความยากหรือหัวข้อ:
- เพิ่ม `type` ใหม่ใน `genUnitX()` ของหน่วยนั้น แล้วให้คืน `input.type` ที่รองรับ
- ถ้าต้องการ input แบบใหม่ ให้เพิ่ม `renderXxx()` และผูกใน `renderInput()` โดยจบด้วยเรียก `finish(q, correct)`
- ระวังตัวเลือกหลอกในโหมด choice ต้องมีคำตอบถูกเพียงข้อเดียวและไม่ซ้ำ
- สำหรับข้อเปรียบเทียบ ให้ถามแบบเจาะจง เช่น “จำนวนใดน้อยกว่า 20 อยู่ 1” แทน “จำนวนใดน้อยกว่า 20”
- มีสคริปต์ตรวจ: extract `<script>` แล้ว `node --check` และ eval generators เพื่อยืนยันคำตอบตรงกับโจทย์

## หลักการออกแบบเกมวิทยาศาสตร์

ไฟล์หลักคือ `ป.1/วิทย/science-quiz-app/index.html` (บน GitHub Pages ใช้ชื่อโฟลเดอร์ `science-quiz/`)

แนวทางที่ตกลงไว้:
- เนื้อหาเป็นความรู้ (ไม่ใช่คำนวณ) จึงใช้ **คลังข้อสอบ** (`lessons[id].bank`) แทนการสุ่มตัวเลข
- แต่ละข้อ `{q, v:อีโมจิ, o:[ตัวเลือก...ข้อแรกคือคำตอบถูก], ex:คำอธิบาย}` — เอนจินสลับตำแหน่งตัวเลือกทุกครั้ง
- แบ่งเป็น 3 บทตามหนังสือ: 1 ตัวเรา, 2 พืช, 3 สัตว์
- โหมดรวม: สุ่มบทละ 7 ข้อ / โหมดบทเดียว: 12 ข้อ
- เน้นแนวข้อสอบ ไม่ถามแค่จำชื่อ: ใส่ข้อ **จำแนกหมวด** (ภายใน/ภายนอก), **หน้าที่**, **เชิงเหตุผล/ประยุกต์** (เช่น พืชไม่มีราก → ตาย), และตัวลวงที่สมเหตุผล
- ใช้อีโมจิเป็นภาพประกอบ (👁️🫀🌱🐦) ไม่พึ่งไฟล์ภาพภายนอก

เวลาจะเพิ่มข้อหรือบท:
- เพิ่ม object ใน `bank` ของบทนั้น (ข้อแรกใน `o` ต้องเป็นคำตอบถูกเสมอ, ตัวเลือกครบ 4 และไม่ซ้ำ)
- ถ้าเพิ่มบทใหม่ ให้เพิ่มใน `lessons` และ `ALL_LESSONS`
- ตรวจด้วย `node --check` และ eval `buildQuiz()` เพื่อยืนยันคำตอบอยู่ในตัวเลือกและไม่ซ้ำ

## หลักการออกแบบเกมคณิตภาษาอังกฤษ (Math in English)

ไฟล์หลักคือ `ป.1/คณิต eng/math-english-app/index.html` (บน GitHub Pages ใช้ชื่อโฟลเดอร์ `math-english/`)

แนวทางที่ตกลงไว้:
- เนื้อหาตามหนังสือคณิตภาษาอังกฤษ (Singapore Math) จำนวน 0–10, บวก-ลบไม่เกิน 10, 7 เรื่อง: Numbers & Words, Comparing, Ordering, Number Bonds, Addition, Subtraction, Word Problems
- **จุดต่างจากคณิตไทยคือศัพท์อังกฤษ** — มีข้อถามคำศัพท์จริง (equal, more than, altogether ฯลฯ) ไม่ใช่แค่คำนวณ
- แต่ละข้อมี `en` (โจทย์อังกฤษ), `say` (ข้อความให้ TTS อ่าน — ใช้คำอ่านตัวเลข เช่น "four plus five"), `th` (คำแปลไทย แสดงใต้โจทย์เสมอ)
- **เสียงอ่านอังกฤษ**: ใช้ `window.speechSynthesis` สำเนียง en-US อ่านอัตโนมัติทุกข้อ + ปุ่ม 🔊 กดซ้ำ — ทำงานในเครื่อง ไม่ต้องต่อเน็ต (ปลุกระบบเสียงด้วย user gesture ตอนกดเริ่ม)
- อังกฤษ TTS อ่านได้ดี (ต่างจากไทยที่ห้ามใช้เพราะเพี้ยน)
- input: `choice` (คำ/ตัวเลข/สัญลักษณ์), `pad` (0–10), `order` (แตะเรียง)

เวลาจะเพิ่มข้อหรือเรื่อง:
- เพิ่ม `type` ใน `genUX()` แล้ว return ผ่าน `item({unit,title,en,say,th,visual,input,answer,ex})`
- `input.options` ข้อแรกไม่จำเป็นต้องเป็นคำตอบ (ระบุ `answer` แยก) เอนจินสลับตำแหน่งให้
- ตรวจด้วย `node --check` + eval generators แกะ `en` มาคำนวณเทียบ `answer`

## ระบบเกม (Gamification) — ใช้ใน grammar และ chinese

เพิ่มความรู้สึกแบบดูโอลิงโก้ เก็บความก้าวหน้าใน `localStorage` (คีย์ `grammar_progress_v1` / `chinese_progress_v1`):
- **XP + เลเวล** (`levelOf` = 100 XP/เลเวล) — ตอบถูก +10 XP (+5 เมื่อสตรีค ≥5), จบชุดเต็ม +30, จับคู่ +30 (ไม่ผิด +20)
- **สตรีครายวัน** (`markDay`) — มาเล่นวันติดกันสะสม, เว้นวันรีเซ็ต
- **เหรียญ (`BADGES`)** — ปลดล็อกตามเงื่อนไข มีป๊อปอัป `showBadgeToast`
- **โหมดจับคู่ (`startMatch`)** — จับคู่คำ↔คำแปล (grammar: pronoun↔ไทย, chinese: 汉字+พินอิน+เสียง↔ไทย) 6 คู่/รอบ
- แถบสถิติบนหน้าแรก (`refreshHome`) โชว์เลเวล/XP/วันติด/เหรียญ
- รูปช่วยจำ: ใช้อีโมจิขนาดใหญ่ (ฝังในไฟล์ ไม่ต้องโหลดจากเน็ต) — จีนมี `HANZI_EMOJI` โชว์บนการ์ดคำรูปธรรม (ComfyUI/Photoshop ไม่ได้ใช้ เพราะ ComfyUI ไม่ได้รันและ Photoshop สั่งอัตโนมัติไม่ได้)

## หลักการออกแบบเกมคำศัพท์ภาษาไทย

ไฟล์หลักคือ `thai-vocab/index.html`

แนวทางที่ตกลงไว้ (วิธีเล่นเหมือนเกมคำศัพท์อังกฤษ แต่ไม่ใช้เสียงอ่านไทย):
- ใช้รูปแบบเดิม: โหมดง่ายมาก / ง่าย / ยาก, popup ดูคำก่อน, ปุ่มดูคำอีกครั้ง, คำใบ้ 2 ครั้ง, โหมดยากไม่มีคำใบ้
- เกมภาษาไทยต้องสอน **วิธีพิมพ์คำ** ด้วย ไม่ใช่แค่เรียงตัวอักษร: สระหน้า `เ แ โ ใ ไ` ต้องกดก่อนพยัญชนะ และสระ/วรรณยุกต์บนล่างต้องเป็นปุ่มแยก
- **ห้ามใช้ Text-to-Speech ภาษาไทย** ในเกมสะกดคำ เพราะ browser อ่านคำไทยบางคำเพี้ยนหรือสะกดทีละตัว
- ถ้ามีเสียงคนอ่านจริง ให้ตัดเป็นไฟล์ย่อยในโฟลเดอร์ `audio/` เช่น `w01.mp3` ถึง `w25.mp3` แล้วใส่ `audio:"audio/w01.mp3"` ใน object ของคำนั้น
- เสียงตอบถูก/ผิด/จบเกม/คำใบ้ของภาษาไทยใช้ไฟล์คนอ่านจริงใน `audio/ui/` แล้วเรียกผ่าน `playUiRecording()` ห้ามย้อนกลับไปใช้ TTS ไทย
- ใช้ภาพ/อีโมจิ + คำใบ้ความหมายแทนเสียง เช่น `💻 คอมพิวเตอร์แบบพกพา` ให้เด็กสะกด `โน้ตบุ๊ก`
- หลังสะกดถูก ให้เลือกภาพหรือความหมายที่ตรงกับคำ เพื่อทวนความหมายแบบเกมอังกฤษ
- แบ่งคำเป็นก้อนใน `chunks` เช่น `["สี","น้ำ","ตาล"]`, `["พืช","สวน","ครัว"]`, `["ไข่","พะ","โล้"]`
- เอนจินแยกปุ่มเป็นตัวสะกดจริงทีละตัว รวมถึงสระ/วรรณยุกต์ เช่น `◌้`, `◌ุ`, `◌็` โดยใช้แนวเดียวกับ repo `thai-word-game`
- โหมดง่ายมากควรโชว์ทั้ง “จำเป็นก้อน” และ “ลำดับการกดปุ่ม” เช่น `โ น ◌้ ต บ ◌ุ ◌๊ ก`
- ตอนแสดงคำที่เด็กกด ใช้ `normalizeThai()` ช่วยจัดลำดับสระ/วรรณยุกต์ให้วางกลับเข้ารูปคำไทย เช่น `โน้ตบุ๊ก`, `สีน้ำตาล`, `ไข่พะโล้`

เวลาจะเพิ่มชุดคำศัพท์ภาษาไทย:
- เพิ่ม object ใน `vocabularyList` ตามรูปแบบ `{w,t,em,chunks,tip}`
- `w` คือคำให้สะกด, `t` คือคำใบ้/ความหมาย, `em` คือภาพจำ, `chunks` คือแบ่งคำให้จำ, `tip` คือทริคหลังตอบถูก
- ถ้าเป็นชุดใหม่คนละสัปดาห์/เลขชุด ควรสร้างโฟลเดอร์ใหม่ เช่น `thai-vocab-no31-35/` แล้วเพิ่มลิงก์ในหน้า `index.html`
- ตรวจด้วย `node --check` หลังแก้ทุกครั้ง

## หลักการออกแบบเกมภาษาจีน (Chinese)

ไฟล์หลักคือ `ป.1/Chinese/chinese-app/index.html` (บน GitHub Pages ใช้ชื่อโฟลเดอร์ `chinese/`)

แนวทางที่ตกลงไว้ (ทำแนวเดียวกับเกมอังกฤษ):
- เนื้อหาตามหนังสือ "ภาษาจีนหรรษา เล่ม 1" 4 บท: ทักทาย&คำลา (你好 再见), คำศัพท์ (我 你 老师 爸爸 妈妈 马 兔 …), พินอิน (สระ a o e / พยัญชนะ / ประสมเสียง), วรรณยุกต์ 4 เสียง (bā bá bǎ bà)
- แต่ละข้อ/การ์ดมี `h` (ตัวจีน hanzi), `p` (พินอิน), `th` (คำแปลไทย), และเสียงอ่าน
- **เสียงอ่านจีน**: `window.speechSynthesis` สำเนียง zh-CN — `speak()` อ่านเฉพาะข้อความที่เป็นตัวจีน (เช็ก `isCJK`) ไม่พยายามอ่านพินอิน/ไทย (จะเพี้ยน); ตัวเลือกที่เป็นตัวจีนมีปุ่ม 🔊 และแสดงพินอินกำกับใต้ (จาก `PINYIN` map)
- โจทย์แบบ "เลือกคำจีน/สถานการณ์" (คำตอบเป็นตัวจีน) จะ **ไม่อ่านอัตโนมัติ** เพื่อไม่เฉลย (ตั้ง `say:""`) ให้เด็กกด 🔊 ฟังทีละตัวเลือกเอง
- ใช้แนว "ง่ายสำหรับเด็กเล็ก": เลือกก่อน–กดยืนยัน, ปุ่มใหญ่, Flashcards เรียนคำก่อน, สตรีค 🔥, ทบทวนข้อผิด (ดู memory [[kids-exercises-easy-for-young-children]])
- ต้องมีฟอนต์จีน (ใช้ฟอนต์ระบบ) และเครื่องต้องมีเสียง zh-CN จึงจะได้ยินเสียงอ่าน (iOS/Android ส่วนใหญ่มี)

## หลักการออกแบบเกมไวยากรณ์อังกฤษ (Grammar)

ไฟล์หลักคือ `ป.1/Grammar/grammar-app/index.html` (บน GitHub Pages ใช้ชื่อโฟลเดอร์ `grammar/`)

แนวทางที่ตกลงไว้:

- **ฟังก์ชันกระตุ้นความจำ (ตามที่ผู้ใช้ขอ):**
  - `missed[]` เก็บข้อที่ตอบผิด → ปุ่ม "ทบทวนข้อที่ตอบผิด" ที่หน้าสรุป เล่นซ้ำเฉพาะข้อผิด (สลับตัวเลือกใหม่)
  - streak/combo (🔥) สะสมตอบถูกติดกัน มีเสียง+ข้อความรางวัลที่ 3/5/10
  - โหมด Flashcards (`FLASH[]`) การ์ดพลิกทบทวนกฎ (active recall) เข้าจากปุ่มบนหน้าแรก
  - เฉลยบอกกฎทุกครั้งที่ตอบผิด
- ใช้แนว "ง่ายสำหรับเด็กเล็ก": เลือกก่อน–กดยืนยัน, เสียงอ่าน 🔊 + คำแปลไทยที่ตัวเลือกสรรพนาม, ปุ่มใหญ่ (ดู memory kids-exercises-easy-for-young-children)
